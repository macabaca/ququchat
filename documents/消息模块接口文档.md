# 消息模块接口文档 (MessageHandler)

本文档详细描述了 `internal/api/handler/message_handler.go` 中的接口逻辑。

**Base URL**: `/api`

---

## 消息对象字段说明

消息相关接口的返回体中，`messages` 数组元素结构如下：

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | string | 消息 ID |
| `room_id` | string | 房间 ID |
| `sequence_id` | int64 | 房间内递增序号 |
| `sender_id` | string | 发送者 ID（可能为空） |
| `content_type` | string | 消息类型：`text` / `image` / `file` / `system` |
| `content_text` | string | 文本内容（仅 text/image 等文本场景） |
| `attachment_id` | string | 附件 ID（文件消息） |
| `parent_message_id` | string | 被引用的父消息 ID（可为空） |
| `parent_sequence_id` | int64 | 被引用父消息在房间内的 sequence_id（可为空） |
| `payload_json` | object | 附件元数据（文件消息） |
| `created_at` | int64 | 创建时间（Unix 秒） |

`payload_json` 在文件消息中对应如下结构：

```json
{
    "attachment_id": "att-uuid",
    "file_name": "report.pdf",
    "mime_type": "application/pdf",
    "size_bytes": 123456,
    "hash": "sha256-...",
    "storage_provider": "minio",
    "created_at": 1678888899
}
```

## 1. 获取指定消息之前的历史记录 (GetHistoryBefore)

用于向上滚动加载更多历史消息（分页）。

- **URL**: `/messages/history/before`
- **Method**: `GET`
- **认证**: 需要 (Header: `Authorization: Bearer <token>`)

### 请求参数
**Query / Form Data**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `message_id` | string | 是 | 参照消息的 ID (cursor) |
| `room_id` | string | 是 | 聊天房间 ID |

### 成功响应示例 (JSON)
```json
{
    "messages": [
        {
            "id": "msg-uuid-1",
            "room_id": "room-uuid",
            "sequence_id": 100,
            "sender_id": "user-uuid",
            "content_type": "text",
            "content_text": "Hello",
            "parent_message_id": "msg-parent-uuid",
            "parent_sequence_id": 88,
            "created_at": 1678888888
        },
        {
            "id": "msg-uuid-2",
            "room_id": "room-uuid",
            "sequence_id": 101,
            "sender_id": "user-uuid",
            "content_type": "file",
            "attachment_id": "att-uuid",
            "parent_message_id": "msg-parent-uuid-2",
            "parent_sequence_id": 95,
            "payload_json": {
                "attachment_id": "att-uuid",
                "file_name": "report.pdf",
                "mime_type": "application/pdf",
                "size_bytes": 123456,
                "hash": "sha256-...",
                "storage_provider": "minio",
                "created_at": 1678888899
            },
            "created_at": 1678888899
        }
    ]
}
```

### 返回情况汇总

| 场景 | HTTP状态码 | 返回内容 (JSON) | 说明 |
| :--- | :--- | :--- | :--- |
| **未登录** | 401 Unauthorized | `{"error": "未登录"}` | Token 缺失或无效 |
| **参数错误** | 400 Bad Request | `{"error": "参数错误..."}` | 缺少 `message_id` 或 `room_id` |
| **房间不存在** | 404 Not Found | `{"error": "房间不存在"}` | `room_id` 无效 |
| **无权访问** | 403 Forbidden | `{"error": "无权查看..."}` | 不是私聊相关方或不是群成员 |
| **参照消息不存在** | 404 Not Found | `{"error": "参照消息不存在"}` | `message_id` 无效 |
| **消息不匹配** | 400 Bad Request | `{"error": "消息不属于该房间"}` | 参照消息不在请求的房间内 |
| **查询成功** | 200 OK | `{"messages": [...]}` | 返回消息列表（按时间倒序排列） |

## 2. 获取指定 Sequence 之后的历史记录 (GetHistoryAfter)

用于客户端增量同步消息（Local-First 架构核心接口）。

- **URL**: `/messages/history/after`
- **Method**: `GET`
- **认证**: 需要

### 请求参数
**Query / Form Data**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `room_id` | string | 是 | 聊天房间 ID |
| `after_sequence_id` | int64 | 否 | 起始 sequence_id (不包含)，默认为 0 |

### 成功响应示例 (JSON)
```json
{
    "messages": [
        {
            "id": "msg-uuid-2",
            "room_id": "room-uuid",
            "sequence_id": 101,
            "sender_id": "user-uuid",
            "content_type": "text",
            "content_text": "World",
            "parent_message_id": "msg-parent-uuid",
            "parent_sequence_id": 88,
            "created_at": 1678888899
        },
        {
            "id": "msg-uuid-3",
            "room_id": "room-uuid",
            "sequence_id": 102,
            "sender_id": "user-uuid",
            "content_type": "file",
            "attachment_id": "att-uuid",
            "parent_message_id": "msg-parent-uuid-3",
            "parent_sequence_id": 100,
            "payload_json": {
                "attachment_id": "att-uuid",
                "file_name": "report.pdf",
                "mime_type": "application/pdf",
                "size_bytes": 123456,
                "hash": "sha256-...",
                "storage_provider": "minio",
                "created_at": 1678888899
            },
            "created_at": 1678888899
        }
    ]
}
```

### 返回情况汇总

| 场景 | HTTP状态码 | 返回内容 (JSON) | 说明 |
| :--- | :--- | :--- | :--- |
| **未登录** | 401 Unauthorized | `{"error": "未登录"}` | Token 缺失或无效 |
| **参数错误** | 400 Bad Request | `{"error": "参数错误..."}` | 缺少 `room_id` |
| **房间不存在** | 404 Not Found | `{"error": "房间不存在"}` | `room_id` 无效 |
| **无权访问** | 403 Forbidden | `{"error": "无权查看..."}` | 不是私聊相关方或不是群成员 |
| **查询成功** | 200 OK | `{"messages": [...]}` | 返回消息列表（按 sequence_id 正序排列） |

## 3. 获取与好友的最新消息 (GetLatestByFriend)

用于初次打开私聊窗口时获取最新消息（不推荐用于同步，建议优先使用 GetHistoryAfter）。

- **URL**: `/messages/history/latest`
- **Method**: `GET`
- **认证**: 需要

### 请求参数
**Query / Form Data**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `friend_id` | string | 是 | 好友的用户 ID |

### 返回情况汇总

| 场景 | HTTP状态码 | 返回内容 (JSON) | 说明 |
| :--- | :--- | :--- | :--- |
| **未登录** | 401 Unauthorized | `{"error": "未登录"}` | Token 缺失或无效 |
| **参数错误** | 400 Bad Request | `{"error": "缺少好友ID"}` | `friend_id` 为空 |
| **非好友关系** | 403 Forbidden | `{"error": "双方不是好友"}` | 数据库中无好友关系记录 |
| **无记录** | 200 OK | `{"messages": []}` | 尚未创建聊天房间或无消息 |
| **查询成功** | 200 OK | `{"messages": [...]}` | 返回最新消息列表（按时间倒序） |

---

**Query / Form Data**:

| :--- | :--- | :--- | :--- |
| `group_id` | string | 是 | 群组 ID |

### 返回情况汇总

| 场景 | HTTP状态码 | 返回内容 (JSON) | 说明 |
| :--- | :--- | :--- | :--- |
